# Product UI Strategy

우리 소싱 솔루션의 서비스 방식과 PWA UI/UX 방향에 대한 기획 메모.

주의사항:

- 이 문서는 기획 기준 문서이며, 실제 코드 반영은 별도 요청 시 진행한다.
- 사용자가 직접 일일이 소싱하는 방식이 아니라, 우리 솔루션이 정기적으로 데이터를 수집/저장/정렬한 뒤 결과를 보여주는 구조를 전제로 한다.

## 서비스 운영 방식

- 우리 솔루션이 정기적으로 네이버 API 기반 데이터를 수집한다.
- 수집된 데이터를 저장하고 정제한 뒤, 내부 기준으로 랭킹을 계산한다.
- 사용자는 직접 검색 노동을 하기보다, 정렬된 소싱 결과를 보고 판단한다.
- Bright Data는 1차 네이버 API 수집 단계가 아니라, 이후 쿠팡 실판매 데이터 크롤링 단계에서 사용한다.

## 제품 방향

- 제품 형태는 `PWA`로 배포한다.
- 사용자 경험은 `넷플릭스형 탐색 UI`를 기본으로 한다.
- 단순 콘텐츠 탐색형 UI가 아니라, 카드 안에 소싱 판단 지표를 함께 노출하는 하이브리드 구조로 설계한다.

핵심 원칙:

- 보는 경험은 넷플릭스처럼 직관적이어야 한다.
- 판단하는 경험은 소싱 대시보드처럼 정보가 있어야 한다.
- 홈 화면은 빠른 탐색, 상세 화면은 깊은 분석에 집중한다.

## 홈 화면 방향

홈은 넷플릭스처럼 `가로 row 중심` 구조로 구성한다.

예시 row:

- 오늘의 TOP 후보
- 조회수 높은 상품
- 경쟁 강도 낮은 상품
- 급상승 후보
- 카테고리별 추천
- 쿠팡 확장 가능성 높은 상품
- 내가 저장한 후보

홈의 목적은 사용자가 많은 후보를 빠르게 훑고 1차 선별하는 것이다.

## 카드 설계 방향

넷플릭스형 카드 레이아웃을 유지하되, 카드 안에 소싱 판단 정보도 같이 보여준다.

카드 기본 구성:

- 상품 이미지
- 상품명
- 카테고리 또는 테마
- 종합 점수
- 조회수 또는 관심도 지표
- 경쟁 강도
- 가격대
- 상태 뱃지

추천 뱃지 예시:

- 신규
- 급상승
- 저경쟁
- 고마진 예상
- 추천

카드 정보 원칙:

- 카드에 너무 많은 텍스트를 넣지 않는다.
- 숫자 지표 2~3개와 뱃지 1~2개 정도로 제한한다.
- 자세한 근거 데이터는 상세 화면으로 넘긴다.

예시 카드 하단 정보:

- 조회 8.2k
- 경쟁 낮음
- 예상마진 24%

## 상세 화면 방향

상세 화면은 탐색보다 `판단`에 집중한다.

상세에서 보여줄 핵심 정보:

- 네이버 기준 상품 기본 정보
- 가격 분포
- 판매처 수
- 리뷰 수 및 평점
- 키워드 연관성
- 카테고리 적합도
- 내부 종합 점수와 점수 근거
- 추후 Bright Data 기반 쿠팡 판매 데이터

상세 화면의 핵심은 `왜 이 상품을 추천했는지 설명할 수 있어야 한다`는 점이다.

## 저장/검토 구조

사용자는 모든 후보를 바로 결정하지 않기 때문에, 보관 중심 기능이 필요하다.

추천 구조:

- 저장한 상품
- 검토 예정
- 제외한 상품
- 재확인 필요

이는 넷플릭스의 `내 리스트`와 유사한 역할을 한다.

## PWA 적합성

이 서비스는 PWA와 궁합이 좋다.

이유:

- 사용자가 매일 짧게 접속해 신규 후보를 확인하기 좋다.
- 앱 설치 부담이 낮다.
- 반복 방문형 서비스에 적합하다.
- 푸시 알림 구조와 잘 맞는다.

예시 알림:

- 오늘 신규 고득점 상품 12개
- 관심 카테고리 급상승 상품 발견
- 경쟁도 낮은 후보 5개 업데이트

## 중요한 전제

이 서비스의 핵심 경쟁력은 단순 UI가 아니라 `랭킹 엔진`에 있다.

중요 설계 요소:

- 점수 기준
- 중복 제거 기준
- 카테고리별 가중치
- 신상품 판단 기준
- 급상승 판단 기준
- 쿠팡 확장 가능성 점수
- 시즌성 보정

즉, 구조는 다음 흐름을 따른다.

1. 백엔드가 주기적으로 수집
2. 데이터 정제 및 중복 제거
3. 내부 점수 계산
4. 사용자에게 랭킹 형태로 제공
5. PWA에서 넷플릭스형 UI로 탐색
6. 상세 화면에서 근거를 보고 판단

## 결론

최종 방향은 다음과 같다.

- 정기 소싱 후 랭킹 제공 방식으로 간다.
- PWA로 배포한다.
- 홈은 넷플릭스형 탐색 UI로 구성한다.
- 카드에는 조회수, 경쟁 강도, 점수 등 핵심 판단 지표를 함께 노출한다.
- 상세는 분석형 화면으로 설계한다.

한 줄 요약:

`넷플릭스처럼 탐색하고, 소싱 대시보드처럼 판단하는 구조`
